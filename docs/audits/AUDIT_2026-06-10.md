## Severity Rubric

- **CRITICAL**: token/auth corruption, data loss, unsafe credential handling
- **HIGH**: likely operational pain, bypassable security control, supply-chain exposure
- **MEDIUM**: meaningful architecture/testing/DX weakness
- **LOW**: cleanup, polish, consistency

## 1. Executive Summary

The codebase is in considerably better shape than its size (≈65k lines under `lib/`, 284 test files, 4.4k tests) would suggest. The storage layer consistently uses atomic write-then-rename with `0o600` modes and retry taxonomies for Windows locks; the OAuth flow validates `state`, binds servers to loopback only, and redacts tokens in logs; the lease/queue refresh coordination is defensive (internal error swallowing in `readJson`/`pruneExpiredArtifacts` is deliberate, not accidental); the test suite is sandboxed (`test/helpers/global-sandbox.ts`) and CI enforces an 80% coverage gate on PRs.

The real findings cluster in four areas, all addressed in the companion PRs (§5):

1. **Predictable temp-file names** — the same `Math.random()`-suffixed staging-path one-liner copy-pasted across 18 files (security + duplication).
2. **Packaging drift** — engines floor below what undici actually requires, a dead `.npmignore`, an unnecessary `typescript` peer dependency.
3. **Supply-chain inconsistency in CI** — three of four workflows used mutable action tags while the fourth already pinned SHAs.
4. **Documentation drift** — bug-report template instructing commands that don't exist, an undocumented shipped command, stale version claims.

The remaining structural debt — the three god-files and the test-mock monolith — is real but requires multi-day refactors; it is mapped with concrete seams in §4 rather than half-executed here.

## 3. Strengths Worth Preserving

- **Atomic persistence discipline**: every storage write stages to a sibling temp file, `fsync`-class care on rename retries, `0o600`/`0o700` modes asserted on POSIX, EBUSY/EPERM/ENOTEMPTY/EAGAIN retry taxonomy aligned across modules.
- **OAuth hygiene**: loopback-only binding, dual `state` validation, PKCE via `@openauthjs/openauth`, token redaction patterns in `lib/logger.ts:30-35` plus key-level redaction in `lib/auth/auth.ts:94-114`.
- **Test isolation**: `global-sandbox.ts` prevents `~/.codex` contamination; real-I/O tests confined to `mkdtemp` dirs; deterministic property tests via `FAST_CHECK_SEED`; `test/documentation.test.ts` (800+ lines) locks docs/CLI coherence — rare and valuable.
- **Runtime modularization already underway**: `lib/runtime/` (40+ focused files), `lib/storage/` and `lib/codex-manager/commands/` show the extraction pattern the remaining god-files should follow.
- **Release/docs governance**: per-release notes, style guide, runbooks, plugin-manifest version sync.

## 4. Refactor Roadmap (deliberately NOT executed in this audit's PRs)

Each item is sized and sequenced so it can become its own reviewable PR. They are ordered by value/risk.

### 4.1 Architecture seams

1. **Finish the `codex-manager.ts` extraction** (3,810 lines; ~25 of ~50 commands already in `lib/codex-manager/commands/`). Extract the ~15 standalone formatters (`stylePromptText`, `formatRateLimitEntry`, `normalizeFailureDetail`, `formatAccountQuotaSummary`, …) into `lib/codex-manager/formatters/`, then convert the `if (command === …)` chains into a command registry map. Blast radius is small: external consumers touch only 4 exports (`runCodexMultiAuthCli`, `autoSyncActiveAccountToCodex`, `styleAccountDetailText`, `formatBackupSavedAt`). Est. 2–3 PRs.
2. **Split `lib/request/fetch-helpers.ts`** (1,497 lines, 30+ exports) along its natural seams: `token-refresh.ts` (`shouldRefreshToken`, `refreshAndUpdateToken`), `error-classification.ts` (`isEntitlementError`, `isWorkspaceDisabledError`, unsupported-model fallback), `url-rewriting.ts` (`extractRequestUrl`, `rewriteUrlForCodex`, proxy resolution), `headers.ts`. Re-export from `lib/request/index.ts` so the public subpath is unchanged. Est. 1–2 PRs.
3. **Carve `runtime-rotation-proxy.ts`** (2,497 lines): server/request-context types → `lib/runtime/rotation-server-types.ts`; stream-failover orchestration → `lib/request/stream-failover-runtime.ts`; rate-limit decision tree → `lib/request/rate-limit-decision.ts`. Only 3 call sites import the module. Est. 2 PRs.
4. **Storage type hygiene**: stop exporting migration internals (`AccountMetadataV1`, `RateLimitStateV3`) from the `lib/storage.ts` facade; create `lib/storage/public-types.ts` and move migrations under an internal path. Only `account-pool-write.ts` and `health.ts` import the internals today. Est. 1 PR.
5. **`lib/ui/` cohesion**: add `lib/ui/index.ts` barrel, rename `copy.ts` → `account-display.ts` (5 importers), split `auth-menu.ts` (724 lines) menu-builder out. Est. 1 PR.
6. **Document the dependency direction** (`types/constants → storage → accounts → runtime → manager/CLI`) and enforce with `eslint-plugin-import`'s `no-cycle`. No cycles exist today; the rule keeps it that way. Est. 1 small PR.

### 4.2 Retry consolidation

Create one `withRetry(operation, { maxAttempts, backoffMs, jitterMs, retryableCodes })` in `lib/fs-retry.ts` (it is the natural home — it already owns `FILE_RETRY_CODES`) and migrate the ~18 divergent loops in `config.ts`, `storage.ts`, `quota-cache.ts`, `recovery/storage.ts`, `codex-manager/commands/uninstall.ts`. Migrate call sites in small batches with their existing tests; do not change per-site attempt counts in the same PR as the mechanical migration. Est. 2 PRs.

### 4.3 Error-contract adoption

Adopt `CodexAuthError`/`CodexNetworkError`/`CodexApiError` in `runtime-rotation-proxy.ts`, `fetch-helpers.ts`, and `config.ts` so `docs/reference/error-contracts.md` is backed by machine-readable types end-to-end. ~8 files. Est. 1–2 PRs.

### 4.4 Testing & CI

1. **Consolidate `ci.yml` + `pr-ci.yml`** into one reusable workflow with a Node 20/22 matrix; keep the PR-only coverage threshold job. Est. 1 PR.
2. **Mock factories for the CLI suite**: extract `test/helpers/cli-test-fixtures.ts`; reduce module-level mocks in `codex-manager-cli.test.ts` from 40+ to ~10. Est. 1 large PR (mechanical).
3. **CLI snapshot tests** for `--help`, `status --json`, `report --json`, `forecast --json` — cheap regression net for output contracts. Est. 1 small PR.

### 4.5 Packaging & distribution

1. **Trim source maps**: set `sourceMap: false`, keep `declarationMap` for library consumers (−~1 MB unpacked, ~39%→~25% of tarball). Verify `check-pack-budget` expectations. 
2. **Expand `config/schema/config.schema.json`** to mirror `PluginConfigSchema` (75+ fields) — ideally generated from the zod schema (e.g. `zod-to-json-schema`) in a build step so it cannot drift again.
3. **`@types/node` policy**: pin to the oldest supported runtime major (e.g. `^20`) or raise engines deliberately; today's `^25` types overpromise.
4. **Lazy postinstall**: move app-bind/launcher detection to first CLI invocation; keep postinstall as a thin no-op-on-CI notice. Removes a whole class of install-time concerns.
5. **Document pin rationale for `undici`** in SECURITY.md (as done for hono/rollup) or relax to `^6.25.0`; consider the undici 7.x line when Node 18 support is dropped.

## 6. Findings investigated and REJECTED (do not re-report)

Recorded so future audits do not re-litigate them; each was claimed by an automated pass and disproven by hand.

| Claim | Why it is wrong |
|-------|-----------------|
| "TOCTOU race in `refresh-lease.ts` `readFreshResult` (`existsSync` then read)" | `readJson` (`lib/refresh-lease.ts:108`) catches all errors and returns null; the `existsSync` is a fast path, not a correctness gate. |
| "Floating promise `void this.pruneExpiredArtifacts()` swallows errors" | The method (`lib/refresh-lease.ts:396`) is fully try/caught internally, including per-entry stat/unlink; fire-and-forget is the design. |
| "EXDEV cross-device rename failure in `quota-cache.ts`" | The temp file is created as a sibling of `QUOTA_CACHE_PATH` (`lib/quota-cache.ts:253`), same directory ⇒ same filesystem; EXDEV cannot occur. |
| "Timeout cleanup scope bug in `refresh-queue.ts` `executeRefresh`" | `timeoutId` is declared before the `try` and cleared in `finally` (`lib/refresh-queue.ts:271,330-334`); `Promise.race` retains handlers on the losing promise, so no unhandled rejection. |
| "Shutdown handler double-registration race" | `ensureShutdownHandler` guard is module-scoped and Node module instantiation is single-threaded; duplicate signal handlers require duplicate module instances, which the package layout does not produce. |
| "Local OAuth callback server needs rate limiting" | Server lives only for the seconds of an interactive login, binds loopback, and first-writer-wins on the callback; a local attacker who can spam it can already do worse. |
