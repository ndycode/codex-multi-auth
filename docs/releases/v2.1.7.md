## v2.1.7 — uninstall command and v2.1.6 lifecycle correction

Honest correction to v2.1.6: the `preuninstall` lifecycle hook shipped there never actually fires on `npm@7+`, which removed those scripts in 2020. The `codex-multi-auth uninstall` command itself works; you just have to run it before `npm uninstall`.

### What's new
- `codex-multi-auth uninstall [--dry-run] [--json] [--clear-accounts]` reverses postinstall: unbinds Codex app rotation, removes OS launchers, strips the plugin entry from `Codex.json`, clears the `node_modules` cache, and conservatively deletes shared `bun.lock` only when no other Codex plugins remain. (#473, #468)

### Fixes
- Removed the dead `preuninstall` entry from `package.json`. The published manifest no longer claims a hook npm will not run. `scripts/preuninstall.js` is preserved and still callable directly via `node scripts/preuninstall.js`. To uninstall fully: run `codex-multi-auth uninstall`, then `npm uninstall -g codex-multi-auth`. (#468)
- `hydrateRuntimeEmails` no longer collapses accounts sharing `accountId === undefined` into the same `Map` entry. (#473)
- `restoreTopLevelModelProvider`/`restoreTopLevelResponseStorage` splice missing keys ahead of the first section header instead of producing invalid TOML. (#473)
- App-bind requires `router.state === "running"` and a live PID before reusing `existingState`, so a dead router cannot trigger a `config.toml` rewrite. (#473)
- `runFix`, `runHealthCheck`, and forecast `saveQuotaCache` paths downgrade transient `EBUSY`/`EPERM` to a partial-success warning via `quotaCacheSaveError`. (#473)
- Dry-run `uninstall` computes the `bun.lock` decision from the actual `Codex.json` plugin list. (#473)
- `--clear-accounts` warns and exits non-zero when no handler is wired. (#473)
- Stream-failover read promise is hoisted before the soft/hard timeout split. (#473)
- `withStorageLock` initializes `releaseLock` as a typed no-op. (#473)
- `removePluginFromList` pre-filters falsy entries with `list.filter(Boolean)`. (#473)
- Dry-run uninstall preview no longer requires `scripts/codex-app-launcher.js` on disk. (#473)

### Other
- Updated `uninstall --help`, `docs/troubleshooting.md`, and the README Repair table to call out the recommended ordering and the `npm@7+` caveat.
- Hardened `Codex.json` read-modify-write paths with `withFileOperationRetry` across the CLI uninstall and the standalone preuninstall script.
- Replaced bare `rmSync` in test cleanup with `removeWithRetry` per `test/AGENTS.md`.
- Validated against lint, typecheck, full vitest suite, package budget, build, and 14 audit checkpoints of end-to-end install/uninstall stress on `npm@11.13.0` (cold-start contention, edge cases, sole/multi-plugin `bun.lock` safety, corrupted `Codex.json` safe-fail, real registry install/uninstall, full CLI surface).
