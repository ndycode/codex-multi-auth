## v2.1.0 — local usage governance and policy controls

Adds a local-only usage ledger, account policies, routing profiles, budget guards, and an optional loopback bridge so you can see and shape how managed accounts are used before requests leave the machine.

### What's new
- Local usage governance: JSONL usage ledger, `codex auth usage`, account policy metadata, routing profiles, budget guards, and `codex auth monitor`. (#448, #449, #450, #451, #454, #455)
- Model and account capability matrix views with runtime policy evaluation before account selection. (#451)
- Optional loopback-only bridge with `/health`, `/v1/models`, `/v1/responses`, hashed client-token storage, and integration snippets. (#451)
- `rotation reset-rate-limits` clears stale runtime pool timers. (#442)

### Fixes
- Wrapped remaining `saveAccounts` call sites with retry handling. (#443)
- Redacted proactive-refresh account emails through `maskEmail`. (#444)
- Plugin-config loading now handles non-Error throws. (#445)

### Other
- Refreshed local governance roadmap and integration guidance. (#447, #452)
- Consolidated `parseBooleanEnv` into one shared helper. (#446)
- Validated against typecheck, lint, doc tests, full test suite, build, and repo hygiene.
