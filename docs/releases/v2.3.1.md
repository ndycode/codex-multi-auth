## History

### Features

- Added `codex-multi-auth history`, a read-only command that lists local Codex sessions by reading the rollout files under `<codex-home>/sessions` (default `~/.codex/sessions`, honoring `CODEX_HOME`) directly. Codex's own `codex resume` view filters threads by the `model_provider` recorded in each session; while runtime rotation or app bind is active that provider is `codex-multi-auth-runtime-proxy`, so sessions created under the native `openai` provider (or vice versa) are hidden from `resume` even though the files are still present. This surfaced as "history not shared across accounts" (#612), but the split is by provider name, not by account — the rollout files are never scattered per account. The command shows every local session regardless of provider, so nothing is hidden. `list` (the default subcommand) prints each session's `updated_at`, `model_provider`, id, thread name, and cwd, most-recent first; `show <id>` prints provider/originator metadata and the first user messages for one session. Both accept `--json`. Reopen any session with `codex resume <id>`.

### Behavior

- `history` is read-only: it makes no network calls and never mutates Codex or multi-auth state. Rollout parsing tolerates malformed or partial JSONL lines, skips files without a `session_meta` record, and treats a missing or unreadable `sessions` directory as an empty listing rather than an error. Filename id-extraction is separator-agnostic so Windows-style rollout paths resolve correctly. A bare leading flag (e.g. `history --json`) is treated as the default `list` form.

## Notes

- Patch release published under the `latest` dist-tag (`npm i -g codex-multi-auth`).
- The provider-filtering behavior in `codex resume` lives in the upstream Codex CLI; to restore its native view, run `codex-multi-auth rotation unbind-app` or `codex-multi-auth rotation disable`. The new `history` command is the way to see all sessions without changing the active provider.
- No runtime-rotation, storage, or auth behavior changed in this release.
