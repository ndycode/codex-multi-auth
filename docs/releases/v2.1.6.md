## v2.1.6 — uninstall command and windows file-lock hardening

Adds an `uninstall` command that reverses every postinstall change, plus broad Windows file-lock hardening across config writes, storage saves, and quota cache paths. (See v2.1.7 for an honest correction about the npm lifecycle hook shipped here.)

### What's new
- `codex-multi-auth uninstall [--dry-run] [--json] [--clear-accounts]` reverses postinstall: unbinds Codex app rotation, removes OS launchers, strips the plugin entry from `Codex.json`, clears the `node_modules` cache, and conservatively deletes shared `bun.lock` only when no other Codex plugins remain. (#473)
- `preuninstall` npm lifecycle script wired to the same cleanup path. (Note: see v2.1.7 — this hook does not actually fire on `npm@7+`.) (#473)

### Fixes
- `hydrateRuntimeEmails` no longer collapses accounts sharing `accountId === undefined` into the same `Map` entry; refreshed tokens land on the originating row. (#473)
- `restoreTopLevelModelProvider`/`restoreTopLevelResponseStorage` splice missing keys ahead of the first section header instead of appending and producing invalid TOML. (#473)
- App-bind no longer rewrites `config.toml` to a dead router's port; the predicate now requires `router.state === "running"` and a live PID. (#473)
- `runFix`, `runHealthCheck`, and forecast `saveQuotaCache` paths downgrade transient Windows `EBUSY`/`EPERM` to a partial-success warning surfaced via `quotaCacheSaveError`. (#473)
- Dry-run `uninstall` and `scripts/preuninstall.js` compute the `bun.lock` decision from the actual `Codex.json` plugin list instead of always reporting "uncertain". (#473)
- `--clear-accounts` no longer silently no-ops when no handler is wired; the CLI warns and exits non-zero. (#473)
- Stream-failover read promise hoisted before the soft/hard timeout split so the chunk that ends a stall is no longer dropped. (#473)
- `withStorageLock` initializes `releaseLock` as a typed no-op so a synchronous Promise constructor throw cannot leave the storage mutex held. (#473)
- `removePluginFromList` pre-filters falsy entries with `list.filter(Boolean)` to match `scripts/install-codex-auth-utils.js`. (#473)
- Dry-run uninstall preview no longer requires `scripts/codex-app-launcher.js` on disk; default loader is deferred behind `!dryRun`. (#473)

### Other
- Hardened `Codex.json` read-modify-write paths with `withFileOperationRetry` (six attempts, exponential backoff with jitter, full Windows error-code set: `EBUSY`/`EPERM`/`EAGAIN`/`ENOTEMPTY`/`EACCES`).
- Replaced bare `rmSync` in test cleanup with `removeWithRetry` per `test/AGENTS.md`.
- Added regression coverage for uninstall, hydrate-emails patching, config-toml splice (with CRLF round-trip), `withStorageLock` mutex release, `saveQuotaCache` warning ordering, the EBUSY retry loop, `--clear-accounts` wiring, preuninstall concurrency, and stream-failover cleanup.
- Validated against lint, typecheck, full vitest suite, package budget, build, and an end-to-end uninstall flow against built `dist/` (eight scenarios).
