## v2.0.2 — device-code login for headless and remote use

Adds `codex auth login --device-auth` as the preferred login path for remote shells and headless boxes where launching a browser is awkward.

### What's new
- New `codex auth login --device-auth` flow with device-code polling and PKCE token exchange through the Codex device redirect URI. (#441)
- Tokens persist through existing account storage, Codex CLI sync, and active-account state. (#441)

### Fixes
- Reject `--device-auth` combined with `--manual` or `--no-browser`; browser-first, manual paste, and no-browser paths are unchanged. (#441)
- Device-code poll responses are kept out of logs and account storage. (#441)

### Other
- Added device-auth coverage for aliases, interval parsing, retries, timeouts, token exchange, non-TTY use, persistence, and log redaction. (#441)
- Validated against typecheck, lint, full test suite, package budget, and dry-run packaging.
