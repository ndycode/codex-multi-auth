## v2.0.1 — runtime rotation on by default

Runtime rotation is now enabled by default for wrapper users, with opt-outs preserved. Install and update flows self-heal the Codex app bind, and the package learned to check for its own updates outside CI environments.

### What's new
- Runtime rotation enabled by default; opt out via `codex auth rotation disable`, `codexRuntimeRotationProxy=false`, or `CODEX_MULTI_AUTH_RUNTIME_ROTATION_PROXY=0`. (#439)
- Install/update self-healing for the packaged Codex app bind plus user-level app launcher routing. (#439)
- Best-effort package auto-update checks for installed users (skipped in CI, tests, and local worktrees). (#439)
- Live quota probes and fallback model defaults updated to current selectors including `gpt-5.3-codex` and `gpt-5.5`. (#439)

### Other
- Validated against typecheck, lint, build, full test suite, and dry-run packaging. (#439)
