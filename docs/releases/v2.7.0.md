# v2.7.0

A minor release that answers "when do I get my quota back?" without leaving the terminal. `codex-multi-auth check` already told you how much of each window was left; it now also tells you when that window resets. Closes [#633](https://github.com/ndycode/codex-multi-auth/issues/633).

This release also carries the product-documentation rewrite that landed after 2.6.1 was cut ([#632](https://github.com/ndycode/codex-multi-auth/pull/632)).

Routing, rotation, storage, the account pool, quota math, and the token flow are unchanged. The reset data was already being fetched and cached — this release only renders it.

## Account health

### Features

- **`check` now prints when each quota window resets**, next to the percentage left in that window ([#633](https://github.com/ndycode/codex-multi-auth/issues/633)):

  ```console
  ✓ Account 1 (...) | live session OK (5h 100%, resets 18:10 | 7d 93%, resets 13:50 on Jul 29)
  ```

  Times use the local system timezone on a 24-hour clock. A reset later today prints as `HH:MM`; past midnight it appends the date. The year is omitted because neither window is longer than seven days.

- **Reset display is scoped to `check`.** The dashboard rows, the account menu, and `forecast` share the same formatters and are byte-identical to 2.6.1. The suffix is opt-in at the formatter level (`CompactQuotaFormatOptions.showReset`), and `check` is the only caller that opts in — so the surfaces with real width pressure keep their compact single-line summaries.

- **Missing and malformed reset data degrades instead of failing.** A window with no reset timestamp keeps its percentage and simply omits the reset clause; a timestamp that is absent, zero, negative, non-finite, or an unparseable date is dropped the same way. Nothing about the account check depends on the reset value being present or valid.

- **Window labels still come from the backend.** They are derived from the reported window length (`5h`, `7d`, …) and fall back to `quota` when the backend omits it, rather than being hardcoded — an account whose headers lack the window length still renders correctly.

### Fixes

- **Quota percentages keep their colour when a reset time is shown.** `styleQuotaSummary` matched each segment against an anchored `label NN%` pattern; appending `, resets ...` broke that anchor and would have dropped every quota segment to the muted tone, silently losing the red / yellow / green severity colouring. The pattern now captures the reset clause and mutes only that part, leaving the percentage toned.

## Docs

### Improvements

- New [`codex-multi-auth check`](../reference/commands.md) reference section covering the output shape, the timezone, the `quota` label fallback, and the missing-timestamp behaviour. It also documents something that was previously undocumented: unlike `fix`, `check` does not skip disabled accounts — it iterates every stored account and re-enables one whose token is still usable.
- Product documentation rewritten against the current architecture, closing the audit's P0/P1 gaps between the docs and the live CLI and config ([#632](https://github.com/ndycode/codex-multi-auth/pull/632)).

## Testing

### Improvements

- Eleven new tests. Nine unit tests in `test/codex-manager-formatters.test.ts` cover the opt-in boundary (default output unchanged), both windows rendering resets, same-day versus later-date formatting, invalid timestamps preserving the percentage, the `showQuotaDetails: false` path, and the colour regression above.
- Two command-level tests in `test/codex-manager-cli.test.ts` drive `auth check` through the real health-check and formatter, mocking only the network probe, and assert the actual printed line. The pre-existing `check` assertions only matched `live session OK` and would have passed whether or not reset times rendered at all.
- Expectations are derived from the same `Intl` calls the formatter uses rather than pinned to an en-US shape, so the suite holds outside en-US.

## Notes

- **The date half of the suffix is locale-dependent.** It is produced by `toLocaleDateString` with an undefined locale, so month text and field order follow the host: `Jul 29` under en-US, `29 Jul` under en-GB, `29. Juli` under de-DE, `7月29日` under ja-JP. The `HH:MM` half is stable. Treat the date shape as presentation, not as an output contract to parse.
- Reset times appear only when `showQuotaDetails` is on, which is the default. Turning it off still reduces the line to a bare `live session OK`.
- `fix --live` prints a similar `live session OK (...)` line and was deliberately left alone; extending it is a candidate follow-up.
- Upgrade with `npm i -g codex-multi-auth`. No state, config, or on-disk layout changes; nothing to migrate, and no action is required for existing installs.
