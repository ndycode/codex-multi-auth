## New Features
- Added a `codex-multi-auth unpin` CLI subcommand that idempotently clears the manual pin set by `codex-multi-auth switch <index>`, bumps the new `affinityGeneration` counter so the runtime rotation proxy invalidates session affinity on the next request, and resumes hybrid rotation through the account pool. Wired through the manager dispatcher and surfaced in canonical CLI help. (#475, #474)
- `codex-multi-auth status` now surfaces the pinned account index and warns when the runtime is currently using a different account than the pin requests, so the divergence is visible during triage. (#475, #474)
- `codex-multi-auth switch <index>` now writes a `pinnedAccountIndex` field that the runtime rotation proxy honors before session affinity, hybrid scoring, and pool fallback. The pinned path is exempt from `markSwitched`/`saveToDiskDebounced`/`syncCodexCliActiveSelectionForIndex` so the proxy never clobbers the pin it is honoring. When the pinned account is rate-limited, cooling down, disabled, or blocked by policy, the proxy returns HTTP 503 with `error.code` = `codex_pinned_account_unavailable` and a `pinnedAccountIndex` field instead of silently rotating. `codex-multi-auth best` clears the pin so the recommended account is honored as the new default. (#475, #474)

## Bug Fixes
- Fixed issue #474: `codex-multi-auth switch <index>` had no effect on the Codex desktop app. The runtime rotation proxy loaded accounts once at startup, ignored `storage.activeIndex`, and overwrote the manual selection on every successful request via `markSwitched("rotation")`. The new pinned-account contract above is the working remediation: `switch` writes `pinnedAccountIndex`, the proxy honors it on every request via a content-hash-keyed per-path read of the storage file. (#475, #474)
- Fixed the second half of issue #474: session affinity locked the Codex desktop app to one account for up to 20 minutes. The desktop app chains chat turns via `previous_response_id`, which the proxy uses as a stable session key. With default-on session affinity, every turn of a chat glued to whichever account first responded — even after `switch`, `unpin`, or `best`. A new `affinityGeneration` counter on `AccountStorageV3` is bumped by `switch`/`unpin`/`best` and observed by the proxy via the same content-hash-keyed read; when the disk generation exceeds the proxy's in-memory tracker, `SessionAffinityStore.clearAll()` runs before `chooseAccount` so the same in-flight request benefits. (#475, #474)
- Fixed `AccountManager.buildStorageSnapshot` silently wiping `pinnedAccountIndex` and `affinityGeneration` on every routine debounced save (rate-limit, cooldown, near-quota refund). The snapshot now refreshes pin/gen from disk just before serializing and only adopts the disk pin when disk `affinityGeneration` strictly exceeds memory, matching the CLI's bump-then-write contract. (#475, #474)
- Fixed concurrent `affinityGeneration` increments across multiple CLI processes. `persistAndSyncSelectedAccount` and `unpin` now re-read the on-disk generation just before save and use `Math.max(inMemory, disk) + 1`, so two concurrent `switch`/`unpin`/`best` invocations cannot lose an invalidation signal and the proxy always sees a strictly increasing generation. (#475, #474)
- Fixed `codex-multi-auth unpin` not using `saveAccountsWithRetry`. Every other mutation path in the codebase wraps storage writes with `saveAccountsWithRetry` (3 attempts, exponential backoff, full Windows error-code set: `EBUSY`/`EPERM`/`EAGAIN`/`EACCES`); `unpin` now matches so a transient Windows file lock cannot silently lose the user's intent. (#475, #474)
- Fixed a synchronous `while (Date.now() < deadline) {}` busy-wait introduced by an earlier hardening pass in `readStorageMetaFromDisk`. The hot-path retry loop is replaced with a single read + per-path content-hash cache fallback; transient errors (`EBUSY`/`EPERM`/`EACCES`/`EAGAIN`/`SyntaxError` from a partial-write rename) preserve the last cached snapshot for that path rather than serving `{ pinnedAccountIndex: null, affinityGeneration: 0 }` defaults that would unnecessarily clear affinity. (#475, #474)
- Fixed `codex-multi-auth status` printing `Pinned: account NaN` or `Pinned: invalid account index <N+1>` when the on-disk pin is corrupt. The guard is tightened to `Number.isInteger`, the raw stored value is shown when the pin is out of range, and a remediation hint pointing at `codex-multi-auth unpin` is included. (#475, #474)
- Fixed module-level singleton state on the runtime rotation proxy's storage cache. The previous single-slot snapshot was shared across multiple proxy instances and concurrent vitest workers pointing at different temp storage files, allowing one path to serve another path's stale pin. The cache is now a `Map<string, StorageMetaSnapshot>` keyed by absolute storage path. (#475, #474)
