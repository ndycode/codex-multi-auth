## v2.1.8 — manual switch reaches the codex desktop app

Fixes issue #474: `codex-multi-auth switch <index>` previously had no effect on the Codex Desktop app because the runtime rotation proxy ignored manual selection and chained chat turns stayed glued to one account for up to 20 minutes. Switch now writes a real pin that the proxy honors, and a new `unpin` command resumes hybrid rotation.

### What's new
- `codex-multi-auth unpin` clears the manual pin set by `switch`, bumps `affinityGeneration` so the proxy invalidates session affinity on the next request, and resumes hybrid rotation. (#475, #474)
- `codex-multi-auth status` surfaces the pinned account index and warns when the runtime is using a different account than the pin requests. (#475, #474)
- `codex-multi-auth switch <index>` now writes `pinnedAccountIndex`; the proxy honors it before session affinity, hybrid scoring, and pool fallback. When the pinned account is unavailable, the proxy returns HTTP 503 with `error.code = codex_pinned_account_unavailable` instead of silently rotating. `best` clears the pin. (#475, #474)

### Fixes
- Runtime rotation proxy now honors manual switch on the Codex Desktop app via a content-hash-keyed per-path read of the storage file; the pinned path is exempt from `markSwitched`/`saveToDiskDebounced`/`syncCodexCliActiveSelectionForIndex`. (#475, #474)
- Session affinity no longer locks the desktop app to one account for 20 minutes; `affinityGeneration` bumps from `switch`/`unpin`/`best` trigger `SessionAffinityStore.clearAll()` before `chooseAccount`, including for the in-flight request. (#475, #474)
- `AccountManager.buildStorageSnapshot` no longer wipes `pinnedAccountIndex`/`affinityGeneration` on routine debounced saves; it refreshes from disk before serializing. (#475, #474)
- Concurrent `affinityGeneration` increments across CLI processes are now atomic via `Math.max(inMemory, disk) + 1`. (#475, #474)
- `unpin` uses `saveAccountsWithRetry` so a transient Windows file lock cannot silently lose the user's intent. (#475, #474)
- Replaced a synchronous busy-wait in `readStorageMetaFromDisk` with a single read plus per-path content-hash cache fallback that preserves the last snapshot on transient errors. (#475, #474)
- `status` no longer prints `Pinned: account NaN` for corrupt pins; the guard tightens to `Number.isInteger`, shows the raw stored value, and points at `unpin`. (#475, #474)
- Runtime rotation proxy storage cache is now a `Map<string, StorageMetaSnapshot>` keyed by absolute path, fixing cross-instance and cross-worker pin leakage. (#475, #474)

### Other
- Documented the new pin contract in `docs/reference/commands.md` and the `codex_pinned_account_unavailable` 503 row in `docs/reference/error-contracts.md`; updated CLI help for `unpin` and `best`.
- Introduced `PersistedSwitchReasonSchema` in `lib/schemas.ts` as the single source for the persist switch-reason union; added `"manual"` to the `markSwitched`/`lastSwitchReason` reasons.
- Added 78 behavioral tests across pin honored, affinity invalidation, end-to-end real `http.Server`, and pin safety suites.
- Validated against lint, typecheck, full vitest suite (3974 tests across 267 files), production build, and live Windows CLI smoke including a real upstream Codex API quota probe.
