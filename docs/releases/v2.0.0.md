## v2.0.0 — runtime responses rotation lands

The 2.0 release introduces a runtime rotation proxy that lets wrapper-launched Codex CLI and app sessions failover across managed accounts without restarts. Configuration moves to a wrapper-first model with reversible Codex app binding and shadow-home isolation.

### What's new
- Opt-in runtime rotation via `codex auth rotation enable|disable|status|bind-app|unbind-app`. (#438)
- New `codex-multi-auth-runtime-proxy` provider path forwards official Codex CLI/app Responses requests. (#438)
- Quota, auth-refresh, network, and upstream failover across managed accounts without session restarts. (#438)
- Shadow `CODEX_HOME` handling, sync-back for refreshed Codex state, reversible app binding, and runtime helper for `codex app`. (#438)

### Fixes
- Proxy no longer leaves stale `content-encoding` on decoded upstream responses. (#438)
- Hardened shadow-home sync locking and kept runtime rotation localhost-only with an authenticated local client key. (#438)

### Other
- Refreshed docs for the 2.0 architecture: routing, app bind, storage paths, troubleshooting, and audit archives. (#438)
- Validated against typecheck, lint, full test suite, dry-run publish, clean-install smoke, and dist-tag checks.
