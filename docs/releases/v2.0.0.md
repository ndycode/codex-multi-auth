# Release v2.0.0

Release line: `stable`

This major release ships official Codex runtime account rotation support and the hardened release fixes from PR #438.

## Scope

- Package version prepared for publish: `2.0.0`
- Previous stable release: `v1.3.1`
- Semver rationale: major bump for the new runtime-rotation release line. The proxy remains opt-in, but the release adds new runtime routing, app helper, and shadow-home sync behavior around official Codex sessions.

## What Changed

### Runtime Rotation Proxy

- added a loopback-only Responses API proxy for official Codex sessions
- added `codexRuntimeRotationProxy` and `CODEX_MULTI_AUTH_RUNTIME_ROTATION_PROXY=1`
- added `codex auth rotation enable|disable|status`
- forwarded Codex CLI sessions can rotate managed accounts between backend requests without restarting the session
- upstream decoded responses now drop stale `content-encoding` metadata before returning to the local Codex client

### Codex App and App Server Support

- wrapper-launched `codex app` and `codex app-server` can use the same runtime rotation path when rotation is enabled
- app helper/router state is written with owner-only files and redacted status payloads
- app-server auth, account, and rate-limit protocol responses surface the local multi-auth identity instead of stale official ChatGPT auth failures
- protocol-proxy cleanup resumes `process.stdin` so retry attempts cannot inherit a paused input stream after child-process backpressure

### Shadow Home Sync Hardening

- shadow `CODEX_HOME` sync-back preserves refreshed `auth.json`, `accounts.json`, and `.codex-global-state.json` across compatibility launches
- lock owner metadata writes retry through transient Windows filesystem failures
- failed lock owner metadata writes remove the just-created lock directory so current-session sync-back is not silently skipped
- concurrent shadow sync-back uses per-file state metadata to avoid dropping later refreshed state
