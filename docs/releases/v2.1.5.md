## v2.1.5 — manual update notices and shadow-home isolation

Replaces automatic update behavior with a manual update notice and isolates Codex goal/log/state SQLite files so wrapper-launched runs start from a clean local runtime.

### What's new
- Manual-only update notices on startup for installed wrappers — no `npm install` or global package mutation. (#465)
- Runtime shadow-home isolation for Codex goal, log, and state SQLite files; user config, sessions, plugins, skills, and multi-auth state are preserved. (#467)

### Fixes
- Removed package-owned auto-update behavior and detached startup update checks from forwarded Codex launch paths. (#465)
- `thread/goal/get` proxy now returns `{ "goal": null }` for recoverable upstream errors and exhausted account pools instead of noisy Codex TUI goal-read errors. (#466)
- Runtime shadow-home startup hardened around large or locked Codex `logs_*.sqlite*`/`state_*.sqlite*` files, stale generated dirs, Windows sidecars, casing, and atomic SQLite mirror cleanup. (#467)
- Stabilized update-notice package-root resolution across local Node versions and repo path layouts.

### Other
- Updated docs to point to the manual update command and remove automatic-update expectations. (#465)
- Expanded wrapper regression coverage for shadow-home SQLite skip/mirror behavior, sidecar fallback warnings, session-index fast paths, and update-notice root handling. (#465, #467)
- Validated against lint, typecheck, repo hygiene, vendor provenance, build, package budget, audit, dry-run publish, full tests, published-install smoke, upgrade smoke, and real Codex CLI stress.
